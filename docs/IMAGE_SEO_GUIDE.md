# Image SEO guide

- Use rights-cleared, accurate product and facility imagery. Generated images are illustrative and cannot prove construction, certification, workforce, factory ownership or exact product appearance.
- Name files descriptively: `training-football-size-5-front.webp`, not `IMG_0042.webp`.
- Use WebP/AVIF where quality permits, retain enough detail for commerce and strip unnecessary metadata.
- Supply intrinsic width/height and realistic responsive `sizes`; prioritize only the true above-the-fold hero.
- Product alt text should identify the exact visible product and useful appearance. Decorative category imagery may use empty alt when adjacent text supplies the same link purpose.
- Do not stuff keywords, repeat captions in alt, or describe details not visible.
- Keep the default social image at a stable absolute URL and produce 1200×630 artwork without tiny text.
- Replace image URLs cautiously; image-search discovery and feed approval benefit from stable, crawlable URLs.
- Record photographer/generator, rights, approval owner, product match and replacement date in the content workflow.

After deployment, check layout shift, mobile crops, image response sizes, feed image policy and production Core Web Vitals.
